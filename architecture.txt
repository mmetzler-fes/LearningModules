================================================================================
  LearningModules – Architekturüberblick
================================================================================

STACK
-----
  Backend  : NestJS 11 (Node.js), alle Routen unter /api
  Datenbank: TypeORM + SQLite (data/database.sqlite), synchronize: true
  Auth     : Passport JWT, 24 h, Secret: JWT_SECRET env || 'secretKey'
  Passwort : crypto.scryptSync, gespeichert als "<salt>:<hash>"
  Frontend : Vanilla JS + HTML, kein Bundler

--------------------------------------------------------------------------------

--------------------------------------------------------------------------------
DATENFLUSS (Login → Quiz)
--------------------------------------------------------------------------------

  1. app.init()
       → LoginView.initLoginScreen()  (Admin-Toggle prüfen)

  2. Benutzer logt ein
       → LoginView.enterApp()
       → App.loadTopics()  +  App.loadExamMode()
       → App.setupNavigation()
       → App.navigateToView('teacher-dashboard' | 'student-topics')

  3. Lehrer öffnet Thema
       → TopicsView: Klick "Module" → ModulesView.openTopicModules(topicId)
       → API: GET /api/topics/:id/modules
       → ModulesView.refresh() → H5pRenderer.renderPreview()

  4. Schüler startet Quiz
       → QuizView._startQuiz(topic)
       → quizState initialisieren
       → QuizView._renderModule() → H5pRenderer.renderPreview(..., {quizMode:true})
       → QuizView._finishQuiz()
       → API: POST /api/results (oder public endpoint)

--------------------------------------------------------------------------------
ROLLEN
--------------------------------------------------------------------------------

  admin    Vollzugriff; sieht Admin-Nav (Benutzerverwaltung, Whitelist/Blacklist)
  teacher  CRUD eigene Themen & Module; sieht Ergebnisse
  student  Lesezugriff auf freigegebene Themen; Quiz-Teilnahme (kein JWT)
